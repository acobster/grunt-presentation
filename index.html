Automating your development workflow with Grunt

Coby Tamayo

What is Grunt?

  • In the beginning: a task-based command line build tool for JavaScript projects
  • Now: The JavaScript Task Runner
  • A general-purpose task runner for automating dev workflows
  • Built on Node.js

What sort of tasks?

  • Well, anything...that can be automated
  • Compilation (CSS preprocessors, CoffeeScript)
  • Concatenation
  • Minification
  • Unit testing
  • Dependency management
  • Documentation

How does it work?

Plugin architecture

  1. Declare dependencies
  2. Describe your workflow

How do I use it?

  • Grunt tasks are just Node.js packages!
    • Declare them in package.json
    • Install them with npm
  • Describe workflow with Grunt
    • Load/register tasks in Gruntfile.js
    • Configure tasks with initConfig()

How do I use it?

$ npm install grunt-plugin --save-dev
					
						
// Gruntfile.js
grunt.loadNpmTasks( 'grunt-plugin' )
						
					
						
// Gruntfile.js
...
plugin: {
  options: {
    foo: 'bar'
  }
},
...
